Try77 Diagram Collection

Spread-only follow-up to Try76: the same distribution-first logic, but adapted from path-loss experts to `delay_spread` and `angular_spread` experts with an explicit near-zero spike component.

1. Try77 Spread Expert System

Overview of the spread pipeline: topology-partitioned experts, height-conditioned Stage A, invalid-target masking for non-finite or negative spread values, and a map decoder that reconstructs delay or angular spread from a spike-plus-tail mixture.

2. Why Try77 Uses A Spike Component

Summarizes the evidence path from the Try76 histogram study to the Try77 architecture: spread targets are not simple Gaussian fields, so the model reserves one constrained near-zero mode and lets the other components explain the tail.

3. Height Conditioning In Try77

Try77 keeps the same height-conditioning mechanism as Try76: a sinusoidal height embedding modulates the encoder and shifts the sample-level mixture before the decoder places pixels. The target is different, but the conditioning path is the same.

4. Expert Split And Routing

Try77 keeps the same topology classification and the same city-holdout split semantics as Try76 and Try75. Only the expert family changes: instead of LoS and NLoS experts, each topology class has a delay expert and an angular expert.
